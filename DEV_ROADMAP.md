# Knowhere-RS Development Roadmap (Non-GPU)

Last updated: 2026-03-09

## Goal

Deliver production-grade non-GPU parity with C++ knowhere while maintaining measurable performance advantage under valid recall constraints.

## Phase Plan

## Phase 1: Contract Closure (P0) — ✅ Closed

Objective:

- Close lifecycle/interface contract gaps across core indexes and FFI.

Milestones (done):

1. Unified interface behavior for Build/Train/Add/Search/RangeSearch/AnnIterator/GetVectorByIds/Serialize/Deserialize.
2. FFI capability matrix aligned with actual runtime behavior.
3. Deterministic error handling for unsupported paths.

Exit criteria:

- No open P0 contract mismatch in `docs/PARITY_AUDIT.md`. ✅

## Phase 2: Behavioral Parity (P1) — ✅ Closed

Objective:

- Align non-GPU behavior with C++ semantics for HNSW/IVF/DiskANN/AISAQ/Sparse/MinHash, including remaining interface-level ABI completeness.

Active scoped tasks:

- ✅ No active P1 parity blocker in current queue snapshot.

Recently closed milestones:

- `PARITY-P1-011`: FFI ABI 元数据契约补齐（`IsAdditionalScalarSupported` / `GetIndexMeta`）已完成，FFI 侧具备统一入口、稳定 JSON contract 与回归覆盖。
- `PARITY-P1-010`: Sparse 高级接口对齐（`create_ann_iterator` / `save` / `load`）已完成，支持矩阵与回归证据已写回审计。 

Closed milestones:

1. HNSW advanced path parity (range/iterator/vector retrieval/serialization behavior).
2. IVF and quantization path consistency (training/add/search edge cases).
3. DiskANN/AISAQ lifecycle and configuration parity.
4. Centralized legality matrix for index/datatype/metric combinations.

Exit criteria:

- Sparse/FFI ABI remaining parity items are either implemented or explicitly constrained with consistent Unsupported semantics and regression coverage.

## Phase 3: Validation and Performance Hardening (P2) — ✅ Closed

Objective:

- Standardize benchmark and regression guardrails for trusted performance reporting.

Closed scoped tasks:

- `OPT-P2-003`: 分层门禁执行入口已固化（`scripts/gate_profile_runner.sh`），并完成 `memory/*_RESULT.json.gate_profile` 映射落地。
- `BENCH-P2-002`: recall-gated baseline 已扩展覆盖 ScaNN/RaBitQ/Sparse，并补齐低可信度条目的 `confidence_explanation`。
- `BENCH-P2-003`: cross-dataset 抽样 artifact 流水线已落地并通过 benchmark_only 门禁。

Phase exit criteria:

- default/full/long-tests gate profile can be executed deterministically via unified entrypoints. ✅
- benchmark outputs cover non-GPU core indexes with confidence tags and reproducibility metadata. ✅
- cross-dataset performance artifacts are generated by repeatable schema-compatible workflows. ✅

## Phase 4: Quantized Parity Polish (P2) — ✅ Closed

Objective:

- Close the last governance-only tail after the main P0/P1/P2 streams closed, now that HNSW-PQ advanced-path semantics and legality governance drift have both been stabilized.

Closed scoped tasks:

- `PARITY-P2-001`: HNSW-PQ 高级接口与持久化语义已收口；`has_raw_data=false`、`get_vector_by_ids` Unsupported、`save/load` Unsupported 现为显式稳定 contract，并具备定向回归证据。
- `OPT-P2-004`: legality/governance 漂移已清理完毕；`Index factory/legality` 在 queue/gap/audit 中统一收敛为 Done。

Success criteria:

- `docs/PARITY_AUDIT.md` 不再保留“仅因历史残留而存在”的 `Partial` 状态。 ✅

## Phase 5: Semantic Fidelity, Production Hardening, Performance Leadership (P3) — 🚧 Active

Objective:

- Move beyond entry-level parity closure and make `knowhere-rs` a production-grade non-GPU replacement for native knowhere.
- Prove not only contract stability, but also semantic fidelity and at least one clear non-GPU performance advantage over native knowhere.

Active scoped tasks:

- `OBS-P3-005`: establish the minimum production observability / runtime-governance baseline. Immediate next slice: define the smallest stable observability/resource contract across build/search/load, trace propagation entrypoints for FFI + gate runner, and a minimal resource-estimation schema (`memory_bytes` / `disk_bytes` / `mmap_supported` / `unsupported_reason`).
- `PERF-P3-004`: establish recall-gated native-vs-rs performance leadership baseline on selected non-GPU paths.

Recently closed milestones:

- `SEM-P3-001`: HNSW / IVF / Sparse / ScaNN 的 `GetVectorByIds` / `HasRawData` 语义尾项已完成 focused 收敛；missing-id、empty-index、lossy-index、reorder/raw-data gate 等边界现已具备可审计回归证据。
- `ABI-P3-002`: FFI metadata / additional-scalar 已从最小稳定摘要提升为逐索引可解释 contract；HNSW / IVF / ScaNN / Sparse 的 capability / unsupported_reason / semantics 字段现已具备 focused FFI 回归覆盖。
- `PERSIST-P3-003`: persistence / deserialize-from-file 语义矩阵与 focused regressions 已闭环；`file_save_load` / `memory_serialize` / `deserialize_from_file` 三类边界已在 queue/audit/capability docs 中统一为 stable contract。

Exit criteria:

- Semantic-conformance gaps are documented and either aligned or explicitly constrained with regression evidence.
- Production-relevant persistence / FFI metadata semantics are stable and auditable.
- At least one core non-GPU path shows repeatable, recall-gated performance leadership over native knowhere.
