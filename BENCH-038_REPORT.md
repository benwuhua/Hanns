## 测试的索引类型

本次基准测试覆盖以下 7 种索引类型：

1. **Flat** - 暴力搜索，召回率基准线
2. **HNSW** - 图索引，高召回率
3. **IVF-Flat** - 聚类 + 暴力搜索
4. **IVF-PQ** - 聚类 + 乘积量化
5. **IVF-SQ8** - 聚类 + 8-bit 标量量化
6. **RaBitQ** - 聚类 + 二进制量化
7. **ScaNN** - 各向异性量化 (需要 scann feature)

## 与 C++ knowhere 对比

> 注：C++ 数据为典型参考值，实际取决于硬件、编译选项和优化级别

| 索引 | Rust QPS | C++ QPS (参考) | Rust R@10 | C++ R@10 (参考) | 性能比 |
|------|----------|----------------|-----------|-----------------|--------|
| Flat | ~20 | 5000+ | ~100% | 100% | ~0.4% |
| HNSW | ~300-500 | 2000-3000 | ~97% | 95-98% | ~15-20% |
| IVF-Flat | ~80-150 | 3000-5000 | ~95% | 85-95% | ~3% |
| IVF-PQ | ~100-200 | 5000-8000 | ~91% | 85-92% | ~2-4% |
| IVF-SQ8 | ~90-180 | 4000-6000 | ~93% | 88-93% | ~2-4% |
| RaBitQ | ~200-400 | 8000-10000 | ~88% | 75-85% | ~3-5% |

**性能差距分析**:

1. **Flat 索引**: Rust 实现 QPS 显著低于 C++，主要原因是：
   - 缺少 SIMD 优化 (AVX2/AVX-512)
   - 内存布局差异
   - 缺少多线程并行搜索

2. **HNSW 索引**: Rust 实现相对接近 C++，约 15-20% 性能：
   - 图遍历逻辑已优化
   - 主要差距在距离计算 SIMD

3. **IVF 类索引**: Rust 实现约 2-5% C++ 性能：
   - 量化解码开销大
   - 缺少 SIMD 加速的距离计算
   - 聚类搜索未充分并行化

4. **RaBitQ**: Rust 实现约 3-5% C++ 性能：
   - 二进制量化解码效率高
   - 主要瓶颈在汉明距离计算

## 关键发现

### 1. 召回率表现
- **Flat**: 100% 召回率，作为基准线
- **HNSW**: 最佳召回率 (>97%)，适合高精度场景
- **IVF-SQ8**: 召回率损失小 (~93%)，优于 IVF-PQ
- **IVF-PQ**: 召回率适中 (~91%)，压缩比高
- **RaBitQ**: 召回率相对较低 (~88%)，但压缩比最高

### 2. 性能表现
- **RaBitQ**: 最高 QPS，适合高吞吐场景
- **IVF-PQ**: 中等 QPS，平衡性能和召回率
- **HNSW**: 中等 QPS，高召回率
- **Flat**: 最低 QPS，不适合大规模实时搜索

### 3. 内存占用
- **RaBitQ**: 最低 (~200MB)，压缩比最高
- **IVF-PQ**: 低 (~300MB)，16 子空间量化
- **IVF-SQ8**: 中 (~400MB)，8-bit 量化
- **Flat/HNSW**: 高 (~500-900MB)，原始向量存储

### 4. 参数影响
- **nprobe**: 增加 nprobe 显著提升召回率，但降低 QPS
- **ef_search**: HNSW 的 ef_search 影响类似，高 ef = 高召回 + 低 QPS
- **最佳平衡点**: nprobe=10-50 或 ef=128-256

## 优化建议

### 短期优化 (1-2 周)
1. **启用 SIMD 优化**: 编译时启用 `--features simd`
2. **并行搜索**: 使用 Rayon 并行化查询处理
3. **距离计算优化**: 针对 L2 距离的 SIMD 实现

### 中期优化 (1-2 月)
1. **量化解码优化**: 减少 IVF-PQ/SQ8/RaBitQ 的解码开销
2. **HNSW 图优化**: 改进图构建算法，减少边数
3. **缓存友好**: 优化内存布局，提升 CPU 缓存命中率

### 长期优化 (3-6 月)
1. **GPU 加速**: 实现 CUDA/Metal 后端
2. **分布式搜索**: 支持多节点并行搜索
3. **混合索引**: 结合多种索引类型的优势
